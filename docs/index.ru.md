# Mqtt — подключение MQTT брокера и привязка топиков

Плагин `Mqtt` добавляет в osysHome интеграцию с MQTT-брокером: он позволяет принимать сообщения с MQTT-топиков, сохранять последние значения и (опционально) обновлять свойства/вызывать методы объектов osysHome. Также поддерживается отправка значений обратно в MQTT при изменении связанных свойств.

## Быстрый старт (для новичка)

### Шаг 1. Настройте подключение к брокеру
1. Откройте админку osysHome и перейдите в модуль `Mqtt` (категория `Devices`).
2. Нажмите кнопку `Settings` (модальное окно).
3. Заполните поля:
   - `Host` — хост MQTT-брокера
   - `Port` — порт (обычно `1883`)
   - `Username` — логин (если требуется)
   - `Password` — пароль (если требуется)
   - `Subscribe topics` — список топиков, на которые брокер должен отправлять сообщения в osysHome (формат: через запятую; пробелы по краям игнорируются)
   - `Auto add topics` — включите, чтобы плагин автоматически создавал записи топиков в своей таблице при первом появлении входящего MQTT-топика
4. Нажмите `Submit`.

После этого в интерфейсе модуля должен появиться статус подключения (иконка/цвет: `connected/connecting/error`).

Важно: создание “Topic” в интерфейсе не означает подписку у брокера. Подписка задаётся именно в поле `Subscribe topics` в `Settings`.

### Шаг 2. Создайте MQTT топик и привяжите его к объекту osysHome
1. На странице `Mqtt` нажмите `Add topic`.
2. Заполните поля:
   - `Name` — удобное имя
   - `Path` — входящий MQTT-топик (точное совпадение с `msg.topic` от брокера)
   - `Path write` — топик для исходящих сообщений (опционально). Если не задан, используется `Path`.
   - `Linked object` — объект osysHome
   - `Linked property` — свойство объекта (опционально). Если заполнено, входящие сообщения будут обновлять это свойство.
   - `Linked method` — метод объекта (опционально). Если заполнено, входящие сообщения будут вызывать этот метод.
   - `Only new value` — обновлять объект/вызывать метод только когда значение изменилось
   - `Readonly` — запретить отправку значений обратно в MQTT при изменении свойства в osysHome
   - `QOS` — QoS (0/1/2)
   - `Retain` — флаг retain при публикации в MQTT
   - `Replace list` — правила замены значений (см. ниже “Опытным”)
3. Нажмите `Submit`.

Топик появится в дереве. У листового узла можно открыть `edit` и `delete`.

### Шаг 3. Проверьте работу на стороне MQTT и osysHome
1. Публикуйте в MQTT сообщение на топик из `Path` (см. сценарии ниже).
2. Проверьте, что в osysHome обновилось значение свойства (например, через `getProperty("ИмяОбъекта.свойство")` в автоматизации/тасках).

## Что поддерживается (входящие и исходящие)

Входящие сообщения:
1. Плагин принимает сообщения из MQTT по подписке `Subscribe topics`.
2. Если в MQTT-топике встречается подстрока `"/set"`, сообщение игнорируется.
3. Если для входящего топика нет записи в таблице и включён `Auto add topics`, запись создаётся автоматически.
4. Далее обновляется запись топика (`value`, `updated`) и выполняются привязки:
   - если задан `Linked method`, вызывается `Объект.Метод`
   - если задан `Linked property`, обновляется свойство `Объект.Свойство`

Исходящие сообщения:
1. Когда изменяется свойство объекта `Объект.Свойство` в osysHome, плагин находит все топики, у которых `linked_object = Объект` и `linked_property = Свойство`.
2. Для каждого такого топика (если `Readonly = false`) плагин публикует новое значение в MQTT в `Path` или `Path write`.
